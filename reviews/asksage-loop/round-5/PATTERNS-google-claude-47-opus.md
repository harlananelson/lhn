## P1 (inaccuracies, contradictions, missing critical facts)
- Section "Notebook Authoring Conventions" → orchestrator example uses `<notebook>.txt` as a positional arg with `--validate`. - Section "Standard notebook setup" imports `F` from `lhn.header` but the bootstrap snippet does not import `F` again — fine — however Pattern 7 and Pattern 4 re-import `from lhn.header import F` inside pattern bodies as if it were not already bound. Either drop the redundant re-imports in Patterns 4/7 or add a note that pattern snippets are standalone. Fix: remove the duplicate `from lhn.header import F` lines in Pattern 4 Step 2 and Pattern 7.
- Pattern 1 Step 4 config example uses `indexFields: [personid]` with a comment "config property is `indexFields`". Confirm against `pipeline_setup` schema — if the actual YAML key is `index_fields` or `indexField` (singular), this propagates as a silent bug because `pipeline_setup` will accept the unknown key and the ExtractItem will fall back to defaults. Fix: cross-check `lhn.bootstrap` / ExtractItem config loader and pin the exact key name; if the loader does strict validation, say so.
- Pattern 3 Step 3 calls `aggregate_fields(... aggfuncs=[F.min, F.max, F.avg, F.count], aggfunc_names=[...])`. `F.count` over a value column counts non-null rows, not distinct patients — fine, but the doc claim "Get min, max, mean per patient (groupby defined by `index`)" omits `count`'s meaning. - Pattern 6 Step 3 join comment says "produces index_diagnosis, last_diagnosis, index_medication, last_medication". If both `dx_index` and `rx_index` carry a `personid` plus any shared columns beyond the join key (e.g., `tenant`), the inner join will produce ambiguous columns and the snippet will fail at runtime. Fix: either select explicit columns before join (`.select('personid', 'index_diagnosis', 'last_diagnosis')`) or note the disambiguation requirement.
- Section "HDL deploy, execute, render, and review" → `render_format.py` auto-detect rule ("md when no graphics/formatted tables, html otherwise"). The harness gate also has opinions about gtsummary/gt — but the doc never tells the author how to *force* a format. Fix: add a one-liner pointing to the `render_format.py` override mechanism (env var or front-matter key) or state "no manual override — detection is authoritative."
- Section 6 (txtarchive `.txt` format): says "The raw cell must be **cell 1**" — good — but doesn't mention that markdown cell numbering must be contiguous starting at 1 and that `txtarchive extract-notebooks` will silently drop out-of-order cells. If that's a real failure mode in the current extractor, add it; if not, ignore.
- Standard notebook setup block: comment says "Bind spark FIRST so spark.sql works even if pipeline_setup raises." This is only true if `from lhn.header import spark` doesn't itself depend on the project context. Confirm `lhn.header` is project-agnostic; if it reads `000-control.yaml` on import, the comment is wrong. Fix: verify and, if `lhn.header` is project-agnostic, leave as-is; otherwise remove the "even if pipeline_setup raises" half-sentence.
- "Use the real API" table/section lists `cacheResult` and `broadcast_flag` as the canonical kwargs (camelCase + snake_case). Pattern 3 Step 2 uses `broadcast_flag=True` — consistent.
